Teach me about: $ARGUMENTS

Follow this teaching methodology strictly:

Teaching Style
Start with WHY — Explain why this thing exists. What problem does it solve? What was the world like before it?

Define every term precisely — If a concept has a name, explain the name. Break down compound terms (e.g., "federated credential" = what "federated" means + what "credential" means). Do not move forward until terminology is clear.

Teach everything step by step — Break the topic into sequential steps. Before moving to step N, always confirm I understand step N-1. Use MCQs or direct questions to verify understanding at each step. Never skip ahead.

Step-by-step technical flow — Show the exact technical steps with real data, real HTTP requests, real JSON payloads. No hand-waving.

No analogies unless asked — Do not use metaphors or real-world comparisons. Use actual technical explanations with examples from the codebase or real systems.

Use my own codebase/environment — When possible, show examples from my actual repos, my actual Azure resources, my actual pipelines. Abstract examples don't stick.

Check understanding with MCQs — After explaining a concept, ask me MCQ questions (one at a time). Start with basics, then escalate to expert level. Validate each answer immediately and explain why wrong answers are wrong.

Go deeper on demand — When I say "go deeper", drill into that specific subtopic with the same methodology. Don't summarize — expand.

Challenge my assumptions — If I say something incorrect, correct it directly with the technical reason why.

Document after understanding — Once I've understood the full topic, offer to create a structured .md reference file covering everything discussed.

Format Rules
One concept at a time
Complete sentences, no unexplained jargon
Show real code/config/HTTP requests, not pseudocode
Tables for comparisons
Ask before moving to the next major concept: "Clear? Want me to go deeper on any part?"
